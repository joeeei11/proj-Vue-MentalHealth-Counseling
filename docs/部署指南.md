## 方式 A：Docker 一键启动（推荐演示用）

> 对方电脑只需安装 Docker Desktop，无需安装 Node.js 或 MySQL。

### A.1 安装 Docker Desktop

下载地址：https://www.docker.com/products/docker-desktop

安装完成后确认 Docker 正在运行（任务栏有鲸鱼图标）。

### A.2 获取项目代码

将整个项目文件夹拷贝到目标电脑，或通过 Git 克隆。

### A.3 创建环境变量文件

在**项目根目录**（`docker-compose.yml` 同级）创建 `.env` 文件：

```env
# MySQL 密码（自定义）
DB_PASSWORD=MySql@2026

# JWT 密钥（任意长随机字符串）
JWT_SECRET=0722feee6f4a0bccefa592ad79728319957c627879695fc5486b20cfcea0259d

# DeepSeek API Key
DEEPSEEK_API_KEY=sk-你的密钥
```

> 可直接复制项目根目录的 `.env.example` 改名为 `.env` 后修改。

### A.4 一键启动

在项目根目录（`docker-compose.yml` 所在目录）打开终端：

```bash
docker-compose up --build
```

**首次运行**会拉取镜像并构建，耗时 3~10 分钟，之后再启动会很快。

看到以下输出表示全部就绪：

```
mental_health_backend   | [server] running on http://localhost:3000
mental_health_frontend  | /docker-entrypoint.sh: Configuration complete
```

### A.5 访问项目

浏览器打开：**http://localhost:5173**

### A.6 停止服务

```bash
# 停止（保留数据）
docker-compose stop

# 停止并清除所有数据（慎用）
docker-compose down -v
```

## 方式 B：手动部署（开发用）

---

## 1. 环境要求

在目标电脑上，确保已安装以下工具：

| 工具 | 最低版本 | 下载地址 |
|------|----------|----------|
| Node.js | 18.x LTS | https://nodejs.org |
| npm | 9.x（随 Node.js 一起安装） | — |
| MySQL | 8.0 | https://dev.mysql.com/downloads/mysql/ |
| Git | 任意版本 | https://git-scm.com |

验证安装：

```bash
node -v       # 应输出 v18.x.x 或更高
npm -v        # 应输出 9.x.x 或更高
mysql --version
git --version
```

---

## 2. 获取代码

将项目代码复制到目标电脑（选择其中一种方式）：

**方式 A：Git 克隆**
```bash
git clone <你的仓库地址>
cd HealthyPlantflom
```

**方式 B：直接复制文件夹**

将整个 `HealthyPlantflom` 文件夹拷贝过去即可。

---

## 3. 配置数据库

### 3.1 启动 MySQL 服务

确保 MySQL 8.0 服务正在运行：

- **Windows**：打开「服务」→ 找到 `MySQL80` → 右键启动
- **macOS**：`brew services start mysql`
- **Linux**：`sudo systemctl start mysql`

### 3.2 创建数据库

以 root 用户登录 MySQL：

```bash
mysql -u root -p
```

执行以下 SQL：

```sql
CREATE DATABASE IF NOT EXISTS mental_health
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

EXIT;
```

### 3.3 导入数据表结构

在项目根目录执行（依次运行，顺序不能颠倒）：

```bash
mysql -u root -p mental_health < database/migrations/001_init.sql
mysql -u root -p mental_health < database/migrations/002_cleanup_corrupted_data.sql
```

执行时会提示输入 MySQL root 密码。

> **说明**：所有 migration 文件均为幂等设计，重复执行不会报错。

## 4. 配置后端

### 4.1 创建环境变量文件

在 `backend/` 目录下，将 `.env.example` 复制为 `.env`：

```bash
cp backend/.env.example backend/.env
```

### 4.2 填写 `.env` 配置项

用文本编辑器打开 `backend/.env`，按下表填写：

```env
# 数据库连接
DB_HOST=localhost
DB_PORT=3306
DB_NAME=mental_health
DB_USER=root
DB_PASSWORD=你的MySQL密码        # ← 必填，改为你本机的 MySQL root 密码

# JWT 鉴权
JWT_SECRET=任意一段长随机字符串    # ← 必填，建议 64 位以上随机字符
JWT_EXPIRES_IN=7d

# DeepSeek AI（AI 对话功能需要）
DEEPSEEK_API_KEY=sk-你的API密钥   # ← 必填，从 DeepSeek 平台获取
DEEPSEEK_BASE_URL=https://api.deepseek.com

# 服务端口
PORT=3000

# 前端地址（用于 CORS 跨域，按实际情况填写）
FRONTEND_URL=http://localhost:5173
```

> **JWT_SECRET 生成方法（任选一种）**：
> ```bash
> # Node.js
> node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
> # 或直接填一段足够长的随机字符串
> ```

> **DeepSeek API Key 获取**：访问 https://platform.deepseek.com → 控制台 → API Keys → 创建新密钥

### 4.3 安装后端依赖

```bash
cd backend
npm install
```

## 5. 配置前端

### 5.1 创建环境变量文件

```bash
cp frontend/.env.example frontend/.env
```

### 5.2 填写 `.env` 配置项

打开 `frontend/.env`，填写后端地址：

```env
VITE_API_BASE_URL=http://localhost:3000
```

> **如果后端部署在远程服务器**，改为服务器 IP 或域名：
> ```env
> VITE_API_BASE_URL=http://192.168.x.x:3000
> ```

### 5.3 安装前端依赖

```bash
cd frontend
npm install
```

需要同时开两个终端窗口：

看到以下输出表示前端启动成功：

## 8. 常见问题

### Q1：后端启动报错 `SequelizeConnectionError`

**原因**：数据库连接失败。

**排查步骤**：
1. 确认 MySQL 服务已启动
2. 确认 `backend/.env` 中 `DB_PASSWORD` 与本机 MySQL 密码一致
3. 确认数据库 `mental_health` 已创建
4. 尝试手动连接验证：`mysql -u root -p -e "show databases;"`

---

### Q2：前端页面空白或接口 404

**原因**：`VITE_API_BASE_URL` 未配置或配置错误。

**解决**：确认 `frontend/.env` 中：
```env
VITE_API_BASE_URL=http://localhost:3000
```
修改 `.env` 后需要**重启前端**（`Ctrl+C` 后重新 `npm run dev`）。

---

### Q3：AI 对话功能无响应

**原因**：`DEEPSEEK_API_KEY` 未填写或无效。

**解决**：
1. 检查 `backend/.env` 中 `DEEPSEEK_API_KEY` 是否正确
2. 确认 DeepSeek 账户余额充足
3. 确认网络可以访问 `https://api.deepseek.com`

---

### Q4：CORS 跨域错误（浏览器控制台报 `Access-Control-Allow-Origin`）

**原因**：后端 CORS 白名单与前端实际地址不匹配。

**解决**：在 `backend/.env` 中添加/修改：
```env
FRONTEND_URL=http://localhost:5173
```
然后重启后端。

---

### Q5：`npm install` 速度慢

切换国内镜像源：

```bash
npm config set registry https://registry.npmmirror.com
```
