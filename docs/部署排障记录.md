# 部署排障记录

## 2026-04-23 首次异地部署

### 环境
- 目标机器：Windows 11，已安装 Docker Desktop
- 部署方式：zip 压缩包 + docker compose

### 部署步骤

1. 将 `HealthyPlantflom_deploy.zip` 解压到 `C:\deploy\HealthyPlantflom`
2. 复制 `.env.example` 为 `.env`，填写以下三项：
   - `DB_PASSWORD`：数据库密码
   - `JWT_SECRET`：JWT 密钥（随机长字符串）
   - `DEEPSEEK_API_KEY`：DeepSeek API Key
3. 清理旧容器和数据卷（全新部署时执行）：
   ```powershell
   docker stop $(docker ps -aq)
   docker rm $(docker ps -aq)
   docker volume prune -f
   ```
4. 启动服务：
   ```powershell
   docker compose up --build -d
   ```
5. 浏览器访问 `http://localhost:5173`

### 遇到的问题及解决方案

#### 问题：预约咨询师时报"服务器内部错误"

**错误日志：**
```
Unknown column 'topic' in 'field list'
Unknown column 'meeting_link' in 'field list'
```

**根本原因：**

`database/migrations/003_add_topic_to_appointments.sql` 中使用了 `ADD COLUMN IF NOT EXISTS` 语法，该语法仅 MariaDB 支持，MySQL 8 不支持，导致 Docker 初始化数据库时该 migration 静默失败，`topic` 和 `meeting_link` 字段未被创建。

**解决方案（已修复源文件）：**

将 `003_add_topic_to_appointments.sql` 中的：
```sql
ADD COLUMN IF NOT EXISTS topic ENUM(...)
```
改为：
```sql
ADD COLUMN topic ENUM(...)
```

**临时手动修复方法（旧版 zip 部署后补救）：**

```powershell
docker exec -it mental_health_mysql mysql -u root -p<DB_PASSWORD> mental_health
```

进入 MySQL 后依次执行：

```sql
ALTER TABLE appointments ADD COLUMN topic ENUM('academic_pressure','relationship_issues','interpersonal','family_issues','career_anxiety','mental_health','other') DEFAULT 'other' AFTER reason;

ALTER TABLE appointments ADD COLUMN meeting_link VARCHAR(500) NULL DEFAULT NULL AFTER cancel_reason;

exit
```

**修复后是否持久：** 是。字段写入 Docker volume，正常重启不会丢失。只有执行 `docker volume prune` 全新部署时才需重新执行。

### 注意事项

- 停止服务用 `docker compose down`（保留数据）
- 全新重置用 `docker compose down -v`（清空数据库）
- 旧版本残留容器会导致端口冲突，全新部署前需先清理
